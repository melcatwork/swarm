# Overall Risk Reduction Percentage - Calculation Explained

**Date**: 2026-04-23  
**Question**: What is the "Overall Risk Reduction %" based on? Attack chains, attack paths, or both?

---

## Short Answer

**BOTH** - The overall risk reduction percentage includes ALL attack chains AND attack paths in `final_paths`:
- ✅ Primary chain (vulnerability-based)
- ✅ Alternate chains (1-4 vulnerability-based)
- ✅ Agent exploration paths (creative scenarios)

---

## Detailed Explanation

### What Goes Into the Calculation

**Step 1: Frontend sends ALL paths to backend**

`frontend/src/pages/ThreatModelPage.jsx` line 609 or 732:
```javascript
const data = await analyzePostMitigation(result.final_paths, mitigationSelections);
```

`result.final_paths` contains:
- **Confirmed vulnerability paths** (source: `'confirmed_vuln_synthesis'`)
  - Primary chain: "Primary chain: CVE-2023-1234 → CVE-2023-5678"
  - Alternate chains: "Alternate-1 chain: ...", "Alternate-2 chain: ...", etc.
- **Agent exploration paths** (source: varies by agent persona)
  - Creative attack scenarios generated by LLM agents
  - May not be grounded in confirmed vulnerabilities

**Step 2: Backend processes EACH path individually**

`backend/app/swarm/mitigations.py` lines 456-616:
```python
for path in attack_paths:  # ALL paths, no filtering
    # Calculate step-by-step mitigation effectiveness
    # Determine path status (neutralized/reduced/viable)
    # Calculate residual CSA risk score for THIS path
```

**Step 3: Backend sums up CSA risk levels from ALL paths**

`backend/app/swarm/mitigations.py` lines 626-648:
```python
# Get original CSA risk level from EVERY path
original_csa_total = sum([
    path.get("csa_risk_score", {}).get("risk_level", 0)
    for path in attack_paths  # ALL PATHS
])

# Get residual CSA risk level from EVERY processed path
residual_csa_total = sum([
    p.residual_csa_risk_score.get("risk_level", 0)
    for p in post_mitigation_paths  # ALL PATHS
    if p.residual_csa_risk_score
])

# Calculate percentage reduction
risk_reduction = ((original_csa_total - residual_csa_total) / original_csa_total * 100)
```

## Example Calculation

### Scenario: 6 Total Paths

**Confirmed Vulnerability Paths** (3 paths):
1. Primary chain: "CVE-2023-1234 → CVE-2023-5678"
   - Original: 20/25 (Very High)
   - Residual: 5/25 (Medium)

2. Alternate-1 chain: "CVE-2023-1234 → CVE-2023-9999"
   - Original: 15/25 (High)
   - Residual: 3/25 (Low)

3. Alternate-2 chain: "CVE-2023-5555 → CVE-2023-5678"
   - Original: 18/25 (High)
   - Residual: 4/25 (Medium)

**Agent Exploration Paths** (3 paths):
4. "Insider Threat + Social Engineering" (Agent: Hacktivist)
   - Original: 12/25 (Medium-High)
   - Residual: 8/25 (Medium-High)

5. "Supply Chain Compromise" (Agent: Nation State)
   - Original: 25/25 (Very High)
   - Residual: 15/25 (High)

6. "Misconfiguration Exploitation" (Agent: Script Kiddie)
   - Original: 10/25 (Medium-High)
   - Residual: 2/25 (Low)

### Calculation

**Original CSA Total**:
```
20 + 15 + 18 + 12 + 25 + 10 = 100
```

**Residual CSA Total**:
```
5 + 3 + 4 + 8 + 15 + 2 = 37
```

**Risk Reduction**:
```
(100 - 37) / 100 × 100 = 63%
```

**Result**: Overall Risk Reduction = **63%**

This percentage includes contribution from:
- ✅ Primary chain (75% reduction: 20→5)
- ✅ Alternate-1 chain (80% reduction: 15→3)
- ✅ Alternate-2 chain (78% reduction: 18→4)
- ✅ Agent path 1 (33% reduction: 12→8)
- ✅ Agent path 2 (40% reduction: 25→15)
- ✅ Agent path 3 (80% reduction: 10→2)

## What This Means for Users

### Chains vs Paths Distinction

In the UI, you see:
- **"Confirmed Vulnerability Paths"** section (collapsed/expandable)
  - Includes: Primary chain + Alternate chains (1-4)
  - Source: Vulnerability intelligence + chain assembly
  - Grounded in actual CVEs and confirmed findings

- **"Agent Explorations - X paths"** section (collapsed/expandable)
  - Includes: Creative attack scenarios from LLM agents
  - Source: Agent personas (Nation State, Hacktivist, etc.)
  - May not be grounded in confirmed vulnerabilities

### Both Are Included in Overall %

When you apply mitigations and see:
```
Overall Risk Reduction: 63%
```

This percentage reflects the aggregate reduction across:
- ✅ ALL confirmed vulnerability paths (primary + alternates)
- ✅ ALL agent exploration paths

**Why?** Because all paths represent potential attack vectors. The overall risk reduction tells you how much your mitigations reduce the **entire attack surface**, not just one subset.

## How It's Different Per Run Type

### Full Swarm Pipeline
- **Confirmed paths**: 1 primary + 4 alternates (if vulns available)
- **Agent paths**: ~5-10 paths from multiple agent personas
- **Overall %**: Includes all ~10-15 paths

### Quick Run (2 Agents)
- **Confirmed paths**: 1 primary + 4 alternates (if vulns available)
- **Agent paths**: ~2-3 paths from 2 exploration agents
- **Overall %**: Includes all ~7-8 paths

### Single Agent
- **Confirmed paths**: 1 primary + 4 alternates (if vulns available)
- **Agent paths**: 1-2 paths from selected agent persona
- **Overall %**: Includes all ~6-7 paths

### Stigmergic Swarm
- **Confirmed paths**: Not applicable (different structure)
- **Agent paths**: Paths from stigmergic exploration with shared graph
- **Overall %**: Includes all paths from stigmergic result

---

## Why Include Both?

**Security Perspective**:
- Vulnerability-based chains show **known, confirmed attack vectors**
- Agent-generated paths show **creative, emergent attack scenarios**
- Defenders need to mitigate **both** to comprehensively reduce risk

**Example**:
- Primary chain exploits CVE-2023-1234 (8.9 CVSS)
- Agent path "Insider Threat" doesn't need a CVE but is equally dangerous
- Overall % shows: "If I implement these mitigations, how much safer am I **overall**?"

---

## Key Takeaways

1. **What's included**: ALL paths in `final_paths` (chains + agent paths)
2. **Calculation method**: Sum of CSA risk_levels (not composite_scores)
3. **Weighting**: Each path contributes proportionally to its CSA risk level
4. **Purpose**: Show aggregate risk reduction across entire attack surface
5. **User control**: Can view per-path breakdown to understand which paths were most/least mitigated
